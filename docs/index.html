Hướng dẫn cài đặt & cấu hình

Tài liệu kỹ thuật để cài đặt, cấu hình và vận hành Chatbot RAG Thủ tục hành chính công (kotaemon + Azure OpenAI + React) trên máy mới — từ clone đến khi chạy được, ~15 phút.

Toàn bộ code (kể cả thư viện kotaemon đã chỉnh) đã nằm trong repo (vendor ở app/libs/). Bạn không cần clone kotaemon riêng — chỉ cần: cài deps → tải index → điền .env → chạy.

Giới thiệu

Chatbot trả lời câu hỏi về thủ tục hành chính (hồ sơ, lệ phí, trình tự, thời hạn, cơ quan thực hiện…) dựa trên kho 5.208 thủ tục lập chỉ mục theo kiến trúc RAG (Retrieval-Augmented Generation). Mỗi câu trả lời chèn trích dẫn 【n】 dẫn về đúng đoạn nguồn.

Sau khi cài, máy có hai phần tách biệt: repo code (git) và index dữ liệu (ngoài git, tải từ HuggingFace).

1 Clone & tạo môi trường

git clone <url-repo> "du-an"
cd "du-an"

python -m venv .venv
.venv\Scripts\python.exe -m pip install -U pip uv

Đường dẫn phải ASCII. Đặt repo ở thư mục tên thuần ASCII (không dấu tiếng Việt) — hnswlib không tạo được file .bin nếu đường dẫn có ký tự non-ASCII.

2 Cài dependencies

Dùng uv (không dùng pip — pip dễ kẹt resolver với deps cũ của kotaemon):

.venv\Scripts\uv.exe pip install --python .venv\Scripts\python.exe `
  --constraint constraints.txt `
  -e "app/libs/kotaemon" -e "app/libs/ktem" `
  fastembed "onnxruntime<1.20" "unstructured>=0.15.8,<0.16" tabulate cachetools

Dấu ` cuối dòng là ký tự nối dòng của PowerShell — có thể gộp thành một dòng.

3 Cấu hình .env

copy .env.example .env

Mở .env và điền giá trị Azure (xem tham chiếu đầy đủ):

AZURE_OPENAI_ENDPOINT=https://<resource>.openai.azure.com/
AZURE_OPENAI_API_KEY=<key>
AZURE_OPENAI_CHAT_DEPLOYMENT=gpt-4o
AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT=text-embedding-3-large

# Đã có sẵn — KHÔNG đổi (xem lưu ý path ASCII):
KH_APP_DATA_DIR=C:\ktem_data

# Tuỳ chọn — bật web fallback cho ReAct agent (để trống = tắt):
BRAVE_API_KEY=<key>

# Tuỳ chọn — bật tính năng "Địa điểm xử lý + bản đồ" (để trống = ẩn tính năng):
SERPAPI_KEY=<key tại https://serpapi.com/ — free ~100 search/tháng>

Không commit .env. File này chứa API key thật và đã được liệt kê trong .gitignore.

4 Tải index đã embed

Không cần crawl/embed lại — tải index dựng sẵn (~728 MB nén) rồi giải nén vào C:\ktem_data:

.venv\Scripts\python.exe scripts\init_index.py --hf-repo MinhTriet/dvc-rag-embeddings

Kiểm tra index bất kỳ lúc nào (in số tài liệu = 5208, kích thước HNSW, trạng thái .env):

.venv\Scripts\python.exe scripts\init_index.py --verify

Có file thủ công: python scripts\init_index.py --from C:\path\to\ktem_index.tar.gz

5 Cài dependencies UI React

cd frontend
npm install
cd ..

6 Chạy ứng dụng

Demo chính là UI React — cần 2 tiến trình song song, mỗi cái một terminal, từ gốc repo.

.venv\Scripts\python.exe -m uvicorn app.api.main:app --port 8000
# Swagger: http://127.0.0.1:8000/docs
cd frontend
npm run dev
# → http://127.0.0.1:5173

Mở http://127.0.0.1:5173. Vite tự proxy /api → :8000 (không lo CORS). Phải chạy backend trước. Có sẵn run-backend.ps1 / run-frontend.ps1 (đã set UTF-8).

Host LAN / build tĩnh

# Terminal 1 — vẫn cần backend
.venv\Scripts\python.exe -m uvicorn app.api.main:app --port 8000
# Terminal 2 — build + preview (port 4173)
cd frontend
npm run build
npm run preview            # → http://<IP-máy>:4173

Bản preview (đã build) nhanh hơn npm run dev nhiều (dev có React StrictMode render gấp đôi + chưa minify).

Xây index từ đầu

# 1. Crawl  → data/raw/*.json (~5208 thủ tục)
.venv\Scripts\python.exe pipeline\crawler\crawl.py
# 2. Parse  → data/corpus/md/*.md
.venv\Scripts\python.exe pipeline\parser\parse.py
# 3. Ingest → C:\ktem_data (song song 10 worker)
.venv\Scripts\python.exe rag\fast_ingest.py --workers 10
# 4. Đóng gói + chia sẻ lên HuggingFace
.venv\Scripts\python.exe scripts\pack_index.py --hf-repo MinhTriet/dvc-rag-embeddings --hf-token hf_xxx

Sau khi ingest lại, phải restart uvicorn để nạp lại vector store vào RAM.

API & SSE

Backend FastAPI mở các endpoint dưới /api. Frontend gọi qua proxy Vite. Xem đầy đủ tại Swagger http://127.0.0.1:8000/docs.

Method · PathMô tả
POST /api/chatGửi tin nhắn, trả SSE (answer / info / citations / done).
POST /api/chat/stopYêu cầu dừng stream (cooperative cancel).
GET /api/conversationsDanh sách hội thoại.
GET /api/conversations/{id}Chi tiết (messages, reasoning, citations).
GET /api/healthKiểm tra trạng thái + số index.

Khung SSE — mỗi event là một dòng data: <json>:

data: {"type": "answer", "text": "..."}        // câu trả lời (trạng thái đầy đủ)
data: {"type": "info", "html": "..."}          // các bước suy luận (HTML)
data: {"type": "citations", "items": [ ... ]}  // danh sách nguồn
data: {"type": "done", "conversation_id": "...", "name": "...", "suggestions": [ ... ]}

Xử lý sự cố

UnicodeEncodeError / charmap khi chạy script in tiếng Việt → console cp1252. Đặt $env:PYTHONUTF8=1 (các run-*.ps1 đã set sẵn).

Mất toàn bộ vector sau ingest (không báo lỗi) → đường dẫn index có ký tự non-ASCII. Giữ KH_APP_DATA_DIR=C:\ktem_data.

FAQ

Có cần GPU không?

Không. LLM & embedding chạy trên Azure OpenAI; máy local chỉ chạy retrieval + UI.

Có bắt buộc Brave API không?

Không. Để trống BRAVE_API_KEY là tắt web fallback; ReAct vẫn tra corpus nội bộ bình thường.

Tính năng "Địa điểm xử lý + bản đồ" cần gì?

Cần đặt SERPAPI_KEY trong .env (free ~100 search/tháng tại serpapi.com); để trống là ẩn tính năng. Sau câu trả lời, bấm "Địa điểm xử lý gần bạn" → định vị (hoặc nhập địa chỉ) → danh sách cơ quan gần nhất kèm bản đồ Leaflet + đường đi (OSRM). Bản đồ, định tuyến và geocode dùng OpenStreetMap, không cần key. Lưu ý: định vị trình duyệt cần localhost hoặc HTTPS — chạy bằng IP LAN thì dùng ô nhập địa chỉ.

Bộ nhớ hội thoại có cần cấu hình không?

Không. Bộ nhớ 2 lớp (tóm tắt ngắn hạn + ký ức dài hạn trên Chroma) mặc định bật, giúp hỏi đáp đa lượt (giải tham chiếu "thủ tục này"). Có thể tắt/bật từng lớp qua MEM_SUMMARY, MEM_EPISODIC, MEM_FACTS (xem tham chiếu .env).

Tốn bao nhiêu dung lượng?

Index ~1.5 GB (ở C:\ktem_data) + deps Python/Node. Tổng ~2.5 GB.

Triển khai online được không?

GitHub Pages chỉ host trang tài liệu tĩnh này. Chatbot thật cần backend FastAPI + index + Azure, nên phải host backend ở nơi khác (VM/Render/Railway) rồi trỏ frontend vào.
