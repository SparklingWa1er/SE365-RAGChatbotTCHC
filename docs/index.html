Chatbot RAG Thủ tục hành chính công Việt Nam

Trợ lý hỏi–đáp về thủ tục hành chính công, dữ liệu từ Cổng Dịch vụ công Quốc gia, xây trên nền kotaemon + Azure OpenAI. Tài liệu này hướng dẫn cài đặt từ máy mới đến khi chạy được, trong ~15 phút.

Bắt đầu cài đặt → Xử lý sự cố
5.208 thủ tục Azure gpt-4o + embedding-3-large React 19 + Vite FastAPI backend Windows · Python 3.10

Giới thiệu

Chatbot trả lời các câu hỏi về thủ tục hành chính (hồ sơ, lệ phí, trình tự, thời hạn, cơ quan thực hiện…) dựa trên kho 5.208 thủ tục đã được lập chỉ mục (RAG — Retrieval-Augmented Generation). Mỗi câu trả lời có trích dẫn 【n】 dẫn về đúng đoạn nguồn.

ℹ️ Điểm mấu chốt

Toàn bộ code (kể cả thư viện kotaemon đã chỉnh) đã nằm trong repo (vendor ở app/libs/). Bạn không cần clone kotaemon riêng — chỉ cần: cài deps → tải index → điền .env → chạy.

Cài đặt (máy mới)

Thực hiện tuần tự 6 bước. Mọi lệnh chạy trong PowerShell, từ thư mục gốc repo.

Clone & tạo môi trường ảo

git clone <url-repo> "du-an"
cd "du-an"

python -m venv .venv
.venv\Scripts\python.exe -m pip install -U pip uv
⚠️ Đường dẫn phải ASCII

Đặt repo ở thư mục tên thuần ASCII (không dấu tiếng Việt). hnswlib không tạo được file .bin nếu đường dẫn có ký tự non-ASCII.

Cài dependencies (dùng uv, KHÔNG dùng pip)

pip dễ kẹt resolver với deps cũ của kotaemon — dùng uv:

.venv\Scripts\uv.exe pip install --python .venv\Scripts\python.exe `
  --constraint constraints.txt `
  -e "app/libs/kotaemon" -e "app/libs/ktem" `
  fastembed "onnxruntime<1.20" "unstructured>=0.15.8,<0.16" tabulate cachetools
ℹ️ Ghi chú

Dấu ` cuối dòng là ký tự nối dòng của PowerShell. Có thể gộp thành 1 dòng nếu muốn.

Cấu hình .env

copy .env.example .env

Mở .env và điền các giá trị Azure:

.env
AZURE_OPENAI_ENDPOINT=https://<resource>.openai.azure.com/
AZURE_OPENAI_API_KEY=<key>
AZURE_OPENAI_CHAT_DEPLOYMENT=gpt-4o
AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT=text-embedding-3-large

# Đã có sẵn — KHÔNG đổi (xem lưu ý path ASCII):
KH_APP_DATA_DIR=C:\ktem_data

# Tuỳ chọn — bật web fallback cho ReAct agent (để trống = tắt):
BRAVE_API_KEY=<key tại https://brave.com/search/api/>

Tải index đã embed từ HuggingFace

Không cần crawl/embed lại — tải index dựng sẵn (~728 MB nén) rồi giải nén vào C:\ktem_data:

.venv\Scripts\python.exe scripts\init_index.py --hf-repo MinhTriet/dvc-rag-embeddings

Kiểm tra index bất kỳ lúc nào (in số tài liệu = 5208, kích thước HNSW, trạng thái .env):

.venv\Scripts\python.exe scripts\init_index.py --verify
💡 Có file thủ công?

Nếu đã tải ktem_index.tar.gz sẵn: python scripts\init_index.py --from C:\path\to\ktem_index.tar.gz

Cài dependencies cho UI React

cd frontend
npm install
cd ..

UI React gọi backend qua tầng FastAPI app/api/. Hợp đồng API + SSE xem frontend/README.md hoặc Swagger tại http://127.0.0.1:8000/docs.

Chạy ứng dụng (2 tiến trình song song)

Demo chính là UI React — cần backend + frontend, mỗi cái một terminal, đều từ gốc repo.

Terminal 1 — Backend (FastAPI)
.venv\Scripts\python.exe -m uvicorn app.api.main:app --port 8000
# Swagger: http://127.0.0.1:8000/docs
Terminal 2 — Frontend (Vite)
cd frontend
npm run dev
# → http://127.0.0.1:5173
ℹ️ Thứ tự

Mở http://127.0.0.1:5173. Vite tự proxy /api → :8000 (không lo CORS). Phải chạy backend trước rồi mới tới frontend.

💡 Có sẵn script tắt

Repo có run-backend.ps1 và run-frontend.ps1 (đã set UTF-8) — chỉ cần .\run-backend.ps1 rồi .\run-frontend.ps1 ở terminal thứ 2.

Nâng cao & tuỳ chọn

Bản host LAN (máy khác truy cập)

Build tĩnh + preview (port 4173)
# Terminal 1 — vẫn cần backend chạy
.venv\Scripts\python.exe -m uvicorn app.api.main:app --port 8000

# Terminal 2
cd frontend
npm run build
npm run preview            # → http://<IP-máy>:4173
💡 Tốc độ

Bản preview (đã build) nhanh hơn npm run dev nhiều — dev có React StrictMode render gấp đôi & chưa minify. Đánh giá tốc độ thật nên dùng preview.

UI Gradio & test headless (phụ)

# UI Gradio (1 tiến trình, không cần API/FE) → http://localhost:7860
.venv\Scripts\python.exe app\app.py

# Test RAG không qua UI
.venv\Scripts\python.exe rag\query_test.py "Hồ sơ đăng ký khai sinh gồm những gì?"

Xây index từ đầu (crawl + embed)

# 1. Crawl  → data/raw/*.json (~5208 thủ tục)
.venv\Scripts\python.exe pipeline\crawler\crawl.py
# 2. Parse  → data/corpus/md/*.md
.venv\Scripts\python.exe pipeline\parser\parse.py
# 3. Ingest → C:\ktem_data (nhanh: 10 worker)
.venv\Scripts\python.exe rag\fast_ingest.py --workers 10
# 4. Đóng gói + chia sẻ lên HuggingFace
.venv\Scripts\python.exe scripts\pack_index.py --hf-repo MinhTriet/dvc-rag-embeddings --hf-token hf_xxx

Xử lý sự cố

⚠️ Lỗi tiếng Việt trên console (UnicodeEncodeError / charmap)

Console Windows mặc định cp1252. Đặt biến trước khi chạy script Python in tiếng Việt: $env:PYTHONUTF8=1. Các script run-*.ps1 đã set sẵn.

⚠️ Mất toàn bộ vector sau ingest (không báo lỗi)

Do đường dẫn index có ký tự non-ASCII. Giữ KH_APP_DATA_DIR=C:\ktem_data — không đặt trong thư mục tên tiếng Việt.

⚠️ Tiến trình ingest treo sau khi kill

Xóa cache theflow rồi chạy lại:

Remove-Item -Recurse -Force "$env:TEMP\claude\theflow_$env:USERNAME"

ℹ️ Linux / HF Spaces

HNSW binary chưa kiểm thử cross-platform. Deploy Linux cần đặt KH_APP_DATA_DIR=/ktem_data và kiểm tra file .bin đọc được.

Câu hỏi thường gặp

Có cần GPU không?

Không. LLM & embedding chạy trên Azure OpenAI (đám mây); máy local chỉ chạy retrieval + UI.

Có bắt buộc Brave API không?

Không. Để trống BRAVE_API_KEY là tắt web fallback; ReAct vẫn tra corpus nội bộ bình thường.

Tốn bao nhiêu dung lượng?

Index ~1.5 GB (ở C:\ktem_data) + deps Python/Node. Tổng ~2.5 GB.

Vì sao “Trả lời nhanh” đôi khi vẫn mất ~1 phút?

Sau khi trả lời, nó còn dựng sơ đồ tư duy (1 lời gọi LLM) + biểu đồ trích dẫn. Có thể tắt 2 addon này cho riêng engine simple để nhanh hơn.

Đổi quy tắc parse/chunk thì sao?

Phải re-ingest rồi đóng gói lại index lên HuggingFace (xem mục Nâng cao) — code repo và index là hai nguồn độc lập.
